## 支柱四：环境即变量

代码必须正确感知运行环境（开发 vs 打包），路径必须动态解析，用户数据必须与应用代码完全隔离。

### DO

- **使用 `app.getPath('userData')` 作为所有用户数据的根目录**，包括数据库、截图、配置文件
- **使用 `app.isPackaged` 或 `is.dev` 判断运行环境**，禁止用 `process.cwd()` 推断环境
- **路径拼接必须使用 `path.join()`**，禁止字符串拼接路径
- **数据迁移逻辑必须有前置条件守卫**：只在旧数据确实存在时才执行迁移，迁移完成后标记已迁移
  ```typescript
  // 正确：有前置条件守卫
  if (fs.existsSync(oldDbPath) && !fs.existsSync(migrationMarker)) {
    performMigration()
    fs.writeFileSync(migrationMarker, Date.now().toString())
  }
  ```
- **打包配置必须显式排除用户数据**：在 `electron-builder.yml` 的 `files` 中排除 `*.db`、`*.db-shm`、`*.db-wal`、`snapshots/` 等
- **资源文件路径必须兼容开发和打包两种模式**：使用 `__dirname` + 相对路径，或 `app.getAppPath()`

### DON'T

- 禁止使用 `process.cwd()` 作为数据存储路径，打包后 cwd 不可预测
- 禁止在代码中硬编码绝对路径
- 禁止将用户数据（数据库、截图、API Key）打包进应用产物
- 禁止在每次启动时无条件执行数据迁移，必须有幂等性保障
- 禁止假设 `__dirname` 在开发和打包环境下指向同一位置
